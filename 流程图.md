# LanLinkChat 项目流程图

下面给出基于当前 Qt 工程整理的项目总体流程图，以及几个主要功能模块的流程图。流程图采用 Mermaid 语法编写，便于后续直接复制到支持 Mermaid 的 Markdown 编辑器、Typora、在线 Mermaid 渲染器或文档平台中生成图片。

## 1. 项目总体流程图

```mermaid
flowchart TD
    A[程序启动] --> B[创建 QApplication 与主窗口]
    B --> C[加载本地配置与历史记录]
    C --> D[启动 PeerManager]
    D --> E[启动 TCP 监听服务]
    D --> F[启动 UDP 广播发现服务]
    D --> G[启动视频解码线程]
    B --> H[初始化界面与信号槽]
    H --> I[初始化媒体设备与视频编码线程]
    F --> J[发现局域网其他节点]
    J --> K[更新联系人列表]
    K --> L{用户操作}
    L --> M[发送文字消息]
    L --> N[创建群聊/发送群消息]
    L --> O[发送文件]
    L --> P[发起视频通话]
    M --> Q[建立或复用 TCP 连接]
    N --> Q
    O --> Q
    P --> Q
    Q --> R[按统一协议封包发送]
    R --> S[对端接收并解析消息]
    S --> T{消息类型判断}
    T --> U[更新聊天记录]
    T --> V[接收文件并保存]
    T --> W[处理通话控制消息]
    T --> X[解码视频帧并显示]
    T --> Y[播放音频数据]
```

## 1.2 答辩 PPT 简洁版总体模块图

```mermaid
flowchart LR
    A[用户界面层\n聊天界面 / 通话界面] <--> B[核心控制层\nPeerManager]
    B <--> C[网络通信层\nUDP发现 / TCP传输 / 协议解析]
    B <--> D[音视频处理层\n摄像头采集 / 视频编解码 / 音频收发]
    B <--> E[数据与存储层\n联系人 / 群聊 / 历史记录]
    C <--> F[局域网其他节点]
    D <--> G[摄像头/麦克风/扬声器]
```

这张图比前面的完整模块图更简洁，适合直接放进答辩 PPT。展示时可以配合一句话说明：系统以 `PeerManager` 为核心，上接界面交互，下连网络通信、音视频处理和本地数据存储，共同完成局域网聊天与视频通话功能。

## 2. 节点发现与基础连接流程图

```mermaid
flowchart TD
    A[程序启动] --> B[创建 DiscoveryService]
    B --> C[绑定 UDP 发现端口 45454]
    C --> D[启动广播定时器]
    D --> E[周期性发送本机公告]
    E --> F[其他节点接收公告]
    F --> G[解析 peerId、昵称、TCP 端口]
    G --> H[更新本地联系人表]
    H --> I{是否需要回复}
    I -- 是 --> J[向发送方回发 reply]
    I -- 否 --> K[等待后续通信]
    H --> L{用户是否发起通信}
    L -- 是 --> M[根据联系人信息建立 TCP 连接]
    M --> N[发送 hello 报文确认身份]
    N --> O[连接注册到 PeerManager]
    L -- 否 --> K
```

## 3. 文字通信模块流程图

```mermaid
flowchart TD
    A[用户选择联系人或群聊] --> B[在输入框输入消息]
    B --> C[点击发送或按 Enter]
    C --> D{当前会话类型}
    D -- 单聊 --> E[调用 sendDirectMessage]
    D -- 群聊 --> F[调用 sendGroupMessage]
    E --> G[封装 direct_message 数据包]
    F --> H[封装 group_message 数据包]
    H --> I[按群成员逐个发送]
    G --> J[建立或复用 TCP 连接]
    I --> J
    J --> K[消息发送到对端]
    K --> L[对端解析消息类型]
    L --> M[触发接收信号]
    M --> N[更新聊天记录]
    N --> O[更新未读消息状态]
    O --> P[保存会话历史]
```

## 4. 文件传输模块流程图

```mermaid
flowchart TD
    A[用户选择文件或拖拽文件] --> B[检查当前联系人是否有效]
    B --> C[打开本地文件]
    C --> D[生成 transferId]
    D --> E[发送 file_begin]
    E --> F[按块读取文件数据]
    F --> G[循环发送 file_chunk]
    G --> H[发送 file_end]
    H --> I[对端收到 file_begin]
    I --> J[创建下载目录与保存文件]
    J --> K[持续接收 file_chunk 并写入]
    K --> L[收到 file_end]
    L --> M{接收大小是否一致}
    M -- 是 --> N[保留文件并提示接收成功]
    M -- 否 --> O[删除不完整文件并提示失败]
```

## 5. 视频通话模块流程图

```mermaid
flowchart TD
    A[用户选择联系人] --> B[点击发起通话]
    B --> C[发送 call_invite]
    C --> D[对端收到邀请]
    D --> E{对端是否接受}
    E -- 否 --> F[结束本次请求]
    E -- 是 --> G[发送 call_accept]
    G --> H[双方进入通话状态]
    H --> I[初始化摄像头与音频设备]
    I --> J[摄像头线程采集本地图像]
    J --> K[主界面接收视频帧]
    K --> L[交给视频编码线程压缩为 JPG]
    L --> M[发送 video_frame]
    M --> N[对端收到视频帧]
    N --> O[交给视频解码线程解码]
    O --> P[更新远端视频显示]
    H --> Q[麦克风采集 PCM 音频]
    Q --> R[发送 audio_chunk]
    R --> S[对端播放音频]
    H --> T{是否结束通话}
    T -- 是 --> U[发送 call_end]
    U --> V[关闭音视频设备并清理界面]
    T -- 否 --> J
```

## 6. 会话历史与未读状态流程图

```mermaid
flowchart TD
    A[收到新消息/文件/通话状态变更] --> B[追加到会话历史]
    B --> C{当前会话是否正在显示}
    C -- 是 --> D[直接刷新聊天窗口]
    C -- 否 --> E[增加未读消息计数]
    D --> F[启动延迟保存定时器]
    E --> F
    F --> G[写入 QSettings]
    G --> H[程序关闭或重启]
    H --> I[重新加载历史记录与未读状态]
```
